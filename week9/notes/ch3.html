Functions

  • Functions are first-class objects that behave the same way as other values.
  • To define a function literal we can use a function declaration.
  • A function can also be declared using the constructor Function(). The body of the function is entered as a string
  • All functions return a value, which can be specified using the return statement, which comes after the return keyword.
  • Every function has a special variable called arguments. We can create a simple function called arguments() that will return the argument sobject so we can see what it contains
  • Arrow Function Arrow functions can be identified by the 'arrow' symbol, => that gives them their name.
    Advantage of using Arrow Functions
    • They are much less verbrose than normal function declarations.
    • Single parameters don't need putting into parentheses.
    • The Body of the function doesn't need placing inside a block if it's only one line.
    • The return keyword isn't required if the return statement is the only statement in the body of the function.
    • They didn't bind their own value of this to the function (we'll see why this is a particulary useful property when we cover objects later in the book).
  • Call Backs A functions can also be given as a parameter to another function. A function that is passed as an argument to another is known as a callback.
  • Array Iterators Arrays have a number of methods that utilize callbacks to make them more flexible.
    Methods
    • forEach()
    • map()
    • reduce()
    • filter()
    • mean()
